Gridit ja flexboksit

Luentomuistiinpanot

Kolmannella viikolla opittiin paljon siitä miten elementit voidaan asetella nettisivuille. Aloitimme flexboxeista, jotka sopivat erityisesti pienempien elementtien asetteluun. Tämän jälkeen siirryimme grideihin, joilla voi tehdä isompia layoutteja. Näitä tuli aika paljon harjoiteltua kolmannella viikolla luennolla pelattujen pelien lisäksi, koska oman nettisivun visiossa asiat eivät ole aivan suorassa ja nätisti allekkain aseteltuna. Piti siis osata soveltaa gridien ja flexboxien eri toimintoja, jotta saisin asioita päällekäin, limittäin, eri tasoihin, offset toisistaan jne.

Opimme myös erilaisia shortcut keys, joilla koodaaminen vscodella helpottuu. Nämä eivät tule vielä luonnostaan, ja kestää ehkä hetken että lihasmuisti oppii käyttämään niitä. Erittäin hyödyllisiä kylläkin.

Paljon muitakin enemmän ja vähemmän sekalaisia vinkkejä jäi mieleen ja muistiinpanoihin. Niitä tässä alhaalla enemmän.

Itselle otetut luentomuistiinpanot

  • flexissä main ja cross akselit, joden suuntaa voi muuttaa. Sen mukaan menee elementtien asettelu. (flex direction: column/row)
  • justify-content:center; /space-around (jokaisen ympärille tilaa) /space evenly /space between /end /start (muuttaa main axis asettelua)
  • align-items:center; (muuttaa cross axis asettelua)
  • rem (root element mitta), fonttikoko on selaimessa automaattisesti 16 px
  • flex ominaisuus menee vain suoraan diviin jossa se määritellään, mutta ei lapsille. lapsille pitää määritellä uudestaan flexiksi
  • flex-grow:1; (kasvata elementin/elementtien kokoa jäljelle olevan tilan mukaisesti)
  • flex-basis sama kuin with mutta akselin mukaisesti
  • flex-wrap: (jos ei mahdu sille riville, menee seuraavalle)
  • align-content:start; (wrappaavat elementit alkuun)
  • grid-template-column: 100px 1fr; (side column on 100 px ja muu sisältö täyttää muun tilan)
  • grid-template-rows: auto 1fr auto; (auto ottaa palkin sisällön ja tekee sen mukaan koon)
  • grid-template-rows: repeat(6, 100px); (toistaa kuusi 100 pikselin kokoista riviä)
  • z-index määrittää millä tasolla elemntti on
  • overflow mahdollistaa, että elementin sisältöä voi rullata
  • developer-toolissa voi valita värin miltä tahansa nettisivulta, hyödyllinen värien kanssa
  • kun haluat yhdistää CSS tiedoston HTML tiedostoon: < link rel="stylesheet" href="./assets/css/mun-css-tiedosto.css">
  • tällä yhdistetää JS HTML tiedostoon < scrip src="">< /script>
  • on eri presettejä, joilla voi nollata tai asettaa parempia aloitusasetuksia kun HTML:ssä on valmiiksi. CSS remedy: preset, miltä aloitustyylit näyttäisivät jos ne keksittäisiin nyt. CSS normalize tekee niin että kaikki selaimet näyttää nettisivut samalla tavalla. Muista, että alimmaisena lisätty css on se jonka määreet jää voimaan jos on samoja määrittelyjä
  • vscodessa -> "view" ja "two columns" voi lisätä ikkunoita vierekkäin, toisessa voi olla html ja toisessa css
  • "render whitespace" voi lisätä välilyönteihin pisteet jotta näkyy paremmin (viewistä)
  • Jos haluaa lisätä sivun sisäisiä linkkejä: < a href="#listan-loppu">Listan loppu< /a>, lisää class-kohtaan/lisäksi elementtiin id="listan-loppu">
  • target="_blank" linkin sisään, niin avaa uudessa välilehdessä
  • transform: scaleX(-1) flippaa objectin, - rotateY(45deg) kiertää

Tehtävästä

Kolmas viikko meni sivun layoutin miettimisessä ja testailussa. Tein kokonaan uuden HTML-tiedoston, johon lisäsin haluamani osat pikkuhiljaa päällekäin. Tämä oli todella hyödyllistä, koska jouduin kertaamaan ja opettelemaan lukemaan koodia tarkemmin, että mikä tekikään mitä ja miten se JS toimikaan. Ekassa prototyypissä oli muutama bugihomma, ja nyt sain kaiken toimimaan. Tekninen toteutus toimii siis tällä hetkellä.

Mitä nyt lisäksi voisi vielä teknisen toteutuksen suhteen kokeillan, olisi lisätä animaatio perhosille ja mahdollisesti käyttäjälle jonkinlaisen "notepadin", jolla tämä voi ottaa muistiinpanoja linnassa tapahtuneen mysteerin selvittämiseksi. Muistiinanot eivät tallennu mihinkään, mutta ehkä loisi hauskaa etsivätunnelmaa?

Sain palaset asettumaan oikeille paikoilleen niin isoilla kun pienemmillä tietokoneen ruuduilla, mutta pienemmillä ei vieläkään ole mitenkään optimaalinen. Tätä pitää myös hieman miettiä. Ehkä haasteena myös mobiililaitteet. Periaatteessa en tiedä onko omasta mielestä tarkoituksenmukaista, että perhosennappaus-mysteeripeliä voisi sellaisenaan pelata kännykällä, mutta voisihan siihenkin mahdollisuuteen tutustua.

Sivua voi tarkastella tästä.

Nyt perhosia spawnaa 10 sekunnin välein, jotta huomio ei kíinnity niihin heti ensimmäiseksi. Ensimmäisestä prototyypistä poikkeavasti nykyisessä pelissä seikkaillaan eri HTML-tiedostojen sisällä, ja peli ei yritä muistaa edellisiä tehtyjä valintoja. Sainsburyn linnassa navigoidaan jokaisen HTML-tiedoston alla löytyvän kartan avulla.

Huvitin itseäni suunnittelemalla ihan oikean mysteerin ja juonen Sainsburyn linnaan. Tarinan on tarkoitus toimia harhautuksena perhosten sieppaamiselle, mutta mysteeri toiminee harhautuksena paremmin jos se ihan oikeasti simuloisi jonkinlaista juonta. Kirjoittamiseen menee tosin yllättävän paljon aikaa, joten ei välttämättä tule mikään mestariteos. Varsinkin kun se ei voi edetä lineaarisesti, vaan sivulla kävijä voi käydä huoneita läpi missä järjestyksessä haluaa. Tuo lisähaastetta mielenkiintoisen jutun kirjoittamiselle, mutta kuitenkin ihan hauska haaste.
