Luentomuistiinpanot

Toisella viikolla mentiin syvemmälle koodiin ja lähdettiin suunnittelemaan nettisivutehtävää. Käytiin myös läpi kuvien optimointia nettisivuja varten. Hauska esimerkki mahdollisimman vihreistä nettisivuista on lowtech magazine, joka toimii aurinkovoimalla ja vain silloin, kun aurinkoenergiaa on saatavilla/akuissa riittää virta.

Alla viikon aikana tehtyjä muistiinpanoja lähinnä itseä varten. Ja siksi myös aika vähän siistityssä muodossa.

Kuvien koon pienentämisestä nettisivuja varten:

  • tinify, tinypng, tiny-img voit pienentää kuvien kokoa
  • fotarissa "save for web legacy" pienentää kuvan viemää tilaa
  • export png as 8bit (vähemmän värejä pienentää kuvan kokoa)
  • webp tiedostomuoto vie suht vähän tilaa
  • jos haluaa rastereilla tehdyn vähän tilaa vievän kuvan: image mode: grayscale, sitten bitmap ja valitse jokin moodi
  • jos kuvassa on paljon pikselimössöä, jpeg muoto on parempi kuin png viedyn tilan suhteen
  • mode: indexed color (jos kuvassa vain muutamaa väriä), sen voi tallentaa fotarissa as smaller file
  • svg tiedostomuoto = tosi pieniä vektorikuvia (voi tehdä illustrattorilla ja kopioida koodin! Sen voi liittää sellaisenaan koodin sekaan.)
  • save as svg -> show code

Koodista:

  • Asioiden keskittäminen HTML + CSS voi olla hankalaa. Tähän vinkkinä: text align, display flex ja flex direction (listojen keskittäminen)
  • Vaihtoehtoisesti: text-align:center; display: grid; place-items:center;
  • CSS= box-model, cascade, inheritance, specifity
  • cascade: tärkeysjärjestys. browser < element < class < id
  • MUTTA elementin sisällä olevat tarkennukset menevät edelle (nostetaan elementtien spesifisyyttä)
  • mitä spesifimmät ohjeet, sitä korkeammalla tärkeysjärjestyksessä
  • ID:n käyttöä ei yleensä suositella, koska sen yliajaminen myöhemmin on hankalaa
  • box-model: elementit paisuvat ulospäin, josa haluaa sisäänpäin marginaalia, lisää bodyyn margin:0; ja * {box-sizing:border-box;} (tähti valitsee kaikki elementit ja antaa niille tyylin)
  • inheritance: tyylimäärittelystä periytyy parent to child ominaisuuksia
  • em -mittayksikköä voi käyttää esimerkiksi jos haluaa jonkin typografisen kaavan fonttien mittayksiköille. mm. gridien ja muiden kanssa voi olla kiva kun kaikki sopii yhteen.
  • CSS tricks on hyvä sivusto vinkeille

Tehtävästä

Ideointivaihe on yleensä yksi hauskimmista vaiheista missä tahansa tehtävässä, mutta samalla myös kuumottavin. Arkistomateriaalin selaaminen sai ekat päivät pään vähän pyörälle, mutta ideoita alkoi pikkuhiljaa syntyä. Inspiroiduin hienista eurooppalaisista linnoista, joissa voi tapahtua vaikka mitä mysteeristä. Varsinkin nyt kun syksy alkaa saapua ja se on parasta aikaa Agatha Christie -tyylisille englatilaisille yläluokan murhamysteereille.

Pelkkä mysteeri ei kuitenkaan innostanut, koska tehtävässä oli tarkoitus kokeilla kaikkea mielenkiintoista ja outoa. Päätin siis tehdä nettisivuista pelilliset, ja idea on oikeastaan harhauttaa nettisivulla kävijää pelin tavoitteesta. Ensimmäisellä silmäyksellä vaikutaa, että pelin tavoite on selvittää jonkin mysteeri, mutta pelin oikeana tarkoituksena onkin kerätä kymmenen perhosta. Pohdin myös mahdollisuutta eräänlaisesta vieraskirjasta ja roolipelittämisestä, koska lapsuudesta muistan kun eräs kamu osasi tehdä nettisivuja, joissa ikään kuin hoidettiin lemmikkejä niin, että alustalla kävijät kirjoittivat tarinoita siitä mitä tekivät lemmikkinsä kanssa. Mutta keskityn aluksi ainakin mysteeriin ja perhosiin.

Visuaalisesti haluan kokeilla kaikenlaisia vähän kummallisia juttuja, esim. että tekee luettavasta tekstistä jotenkin viiston, ikään kuin sitä olisi valokopioitu vähän vinosti paperista. Olisi kiva kokeilla myös jotain filttereitä, joka tekisi tekstiin jotain virheitä, mikä muistuttaisi painokoneen/kirjoituskoneen jälkeä. Haluaisin satukirjamaisen ja jollakin tavalla romanttisen tyylin nettisivujen ulkoasuun, koska se menisi yhteen arkistomateriaalin ja mysteeri tekstiseikkailun kanssa. Vähän mietityttää miten saan yhdistettyä arkistomateriaalia ja svg-kuvia, jotta sivut ei vie liikaa tilaa. Tämän kanssa täytyy ainakin tehdä testejä.

Rakensin perjantaiksi prototyypin tekstiseikkailusta ja perhospelistä, koska halusin ihan ensiksi selvittää onko idea toteutuskelpoinen. Sain kuin sainkin jotenkuten toimivan homman kasaan, joten sitä on nyt kiva lähteä työstämään visuaalisesti.

Prototyyppiä pääsee kurkkimaan tästä.

Chat-gpt:n avulla tehty härveli, mutta suoraan sitä ei voinut kopioida ja vähän kämänen vielä toiminnallisuudeltaan. Esimerkiksi return-toiminnolle chat-gpt ehdotti sen verran epämääräistä ratkaisua, että en halua käyttää sellaisenaan. Sain ehdotuksena, että tarina olisi erillisessä HTML-ikkunassa, koska sitten ei tarvitsisi rakentaa niin monimutkaista choice-puuta(?). Voisi tässä tapauksessa olla ihan hyvä. Sen verran prototyyppi toimii kuitenkin, että pääsen eteenpäin kehittelemään mysteerille jonkinlaista juonta/ideaa ja koristelemaan ulkoasua. Ehkä on vain eri huoneita, joissa voi ihmetellä asioita, ehkä voi jutella henkilöille? Kiinnostaisi myös opetella tekemään animaatioita svg-kuville ja saada perhonen räpyttelemään siipiä.
